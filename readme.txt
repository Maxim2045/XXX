Пароль на авторизации: 1

Это выпускная квалификационная работа студента 4-го курса.
Проект выполнен в среде разработки WPF .NET Framework на языке программирования C#.

Из деканата поступил запрос на программу для автоматического внесения информации о студентах 
в экзаменационные ведомости.

Данные о студентах храняться в Excel таблицах, необходимо перенести в Word шаблон.

Что есть в проекте:

1. Для работы с Excel(копирование) и Word(внесение данных) была использована библиотека OpenXML.
На этом версия для деканата кончается))

2. Имеется реализация окна редактирования Word файлов с помощью контрола RichTextBox. Для переноса информации
из документа на контрол использована библиотека Microsoft.Office.Interop.Word(реализовано в виде copy-paste).

3. Реализованы функции сохранения(FileSteam) и печати(System.Printing) текста с RichTextBox.

4. Выбор шрифта и его размера; подчеркивание, курсив и жирный шрифт.

5. Имеется БД для хранения данных о студентах. Написана в среде SQLite. Создавалась прогрммано с помощью библиотеки 
System.Data.SQLite. Взаимодействие с посредством EntityFramework. Данные о студнтах вносятся в ручную, можно использовать в
качестве списка злостных должников -_-

6. Для шика и лоска(пародирование Win10) были использованы GUI библиотеки FluentWPF и MaterialDesign.

7. Код прокомментирован, исключения обработаны

Имеются попытки в ООП, логика из кнопок вынесена в классы, все связанное с БД то же как будто ООП. 

Ну и если возникают вопросы почему не WPF NET.Core или даже ASP MVC .NET Core, то тут два соответственно объяснения:
1) у заказчика все машина на windows, кроссплатформенность не требуется
2) а на счет web приложения, не захотел разбираться с реализацией диалоговых окон и передаче информации из Word файла 
на какой-нибудь онлайн редактор. Нет дотсаточной информации на эту тему, а для WPF подобные реализации встречались
в рамках обучалок. А так да, API было бы хорошо

Для тех, кто будет работать с схожей темой проект будет полезен.
